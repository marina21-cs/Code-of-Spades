# 0.2 Tooling, Linting & Strict TypeScript

## Context

<context>
With the app running (step 0.1), this step locks in code quality before any feature code exists. The spec's clean-code standards (Section 8 + the implementation guide's clean_code_standards) require TypeScript strict mode, consistent naming, and automated linting/formatting. Setting these up now means every later phase is held to the same bar and the phase checklists can run `tsc` and `lint` as gates.
</context>

<prerequisites>

</prerequisites>

## AI Implementation Prompt

<instructions>
Configure strict TypeScript, ESLint (Expo flat config), and Prettier, and wire up npm scripts.

Think step by step:

1. **Strict TypeScript**
   - In `tsconfig.json`, enable: `strict: true`, `noUncheckedIndexedAccess: true`, `noImplicitOverride: true`, `noFallthroughCasesInSwitch: true`, `forceConsistentCasingInFileNames: true`.
   - Add path aliases under `compilerOptions.paths` so imports use `@/` for the `src/` root (configured fully in 0.3). Example: `"@/*": ["./src/*"]`.

2. **ESLint**
   - Install and initialize ESLint using the Expo ESLint config (`eslint-config-expo`) in flat-config form (`eslint.config.js`).
   - Add `@typescript-eslint` rules for: no unused vars (error, allow `_`-prefixed), no explicit `any` (warn), consistent type imports.
   - Integrate `eslint-plugin-react-hooks` rules (rules-of-hooks error, exhaustive-deps warn).

3. **Prettier**
   - Install Prettier and add a `.prettierrc` (2-space indent, single quotes, trailing commas `es5`, print width 100, semicolons).
   - Add `eslint-config-prettier` so ESLint and Prettier don't conflict.

4. **Scripts**
   - In `package.json`, add scripts: `lint` (`eslint .`), `lint:fix` (`eslint . --fix`), `format` (`prettier --write .`), `typecheck` (`tsc --noEmit`).

5. **Editor + ignore files**
   - Add `.prettierignore` and ensure `.gitignore` covers `node_modules`, `.expo`, `dist`, `*.log`, `.env*`, and native build folders (`android/`, `ios/` if generated by prebuild ‚Äî keep them ignored unless the team commits native dirs).
</instructions>

## Troubleshooting

| Symptom | Likely Cause | Fix |
|---|---|---|
| `@/` imports unresolved | Metro doesn't read tsconfig paths | Add `babel-plugin-module-resolver` or use Expo Router's built-in alias support; restart with `-c` |
| ESLint "parser" errors | Missing `@typescript-eslint/parser` | Install `typescript-eslint` and reference it in the flat config |
| Prettier vs ESLint fighting | Missing `eslint-config-prettier` | Add it last in the ESLint config chain |
| `tsc` floods with library type errors | `skipLibCheck` off | Keep `skipLibCheck: true` (inherited from Expo base) |
